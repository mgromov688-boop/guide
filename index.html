Главный сдвиг: ты режиссёр, не кодер.

90% пишут «сделай красивый лендинг» и получают шаблон. Рабочий подход другой: ты приносишь примеры и задаёшь рамки, а сборку отдаёшь Claude Code. Он видит файлы, сам их правит, сам гоняет проверку. Ты показываешь референс и говоришь «вот так, но по-своему».

Процесс · ядро гайда

Пять шагов от пустой папки до лендинга на домене

Один конвейер. Под каждым шагом — готовый промт: копируй и вставляй в Claude Code.

01

Разведка эталонов

Нельзя собрать «вау», если не видел «вау». Берёшь 3–4 сайта уровня студии и вытаскиваешь по одному сильному ходу на секцию. Готовые блоки можно тянуть прямо в Claude по MCP — живой нейро-фон этой страницы так и появился, из каталога 21st.dev.

→ на выходе: канон + список секций, по одному ходу на каждую
awwwards / godly — насмотренность21st.dev по MCP — готовые блоки с кодом в Claude
промт · разведка + канонкопировать
Вот 3 сайта-эталона (ссылки/скрины). Разбери каждый:
палитра, шрифт, ритм, и какой один приём держит каждую секцию.
Собери из этого мой канон и список секций — по одному ходу на каждую.
Пока ничего не верстай.
02

Канон — рамки проекта

Пока не зафиксируешь палитру, шрифт и ритм — каждый экран получится разным. Канон этой страницы: угольный фон, один оранжевый акцент, Geist для заголовков, моно для кода.

→ на выходе: зафиксированные палитра, шрифт и ритм
my-design — премиум-вкусcalm-design — ритм и воздух
промт · канонкопировать
Зафиксируй канон: 1 фон + 1 акцент, 1 заголовочный + 1 текстовый шрифт,
отступы и ритм. Дальше каждую секцию собираем строго по нему.
03

Сборка — стек премиум-скролла

Плавный скролл, появления по прокрутке, живой WebGL-фон, выпадание плашек. Claude ставит и настраивает сам — тебе важно понимать, что за что отвечает.

→ на выходе: секции собраны по канону, один сильный ход на каждой
Lenis — плавный скроллGSAP ScrollTrigger — эффекты по скроллуWebGL-шейдер — живой фонcontext7 — свежая дока
промт · секциякопировать
Собери секцию строго по канону. Один сильный ход:
живой фон / выпадание плашек / сборка текста.
Сделай чисто, без лишних свечений. Покажи — дальше буду докручивать.
04
Сердце гайда

Гейт качества

Тут рождается разница между «ИИ-поделкой» и премиумом. Проверять инструментами, а не на глаз — новички пропускают именно это, отсюда весь слоп.

→ на выходе: чистый премиум без дизайн-слопа
Impeccable — детектор слопаPlaywright — тест живьёмглиф-чектач-тест 390pxhumanizer — тексты
промт · прогон качествакопировать
Прогони на дизайн-слоп (Impeccable): где перебор эффектов/свечений/шрифтов.
Проверь типографику на уровне глифов и мобилку на 390px реально,
не просто уменьши окно. Дай список правок по приоритету.
05

Деплой на свой домен

Стили статикой, SEO и превью для мессенджеров, бесплатный хостинг. И проверка на проде — «задеплоил» не равно «работает».

→ на выходе: живой лендинг на своём домене
GitHub Pages — хостинг + поддоменOG / SEO — превью ссылки
промт · деплой + проверкакопировать
Вшей стили статикой, добавь SEO/OG/favicon, задеплой на GitHub Pages.
Потом открой боевую ссылку на десктопе и на 390px, поскролль
и подтверди, что всё живо. Не прошёл деплой — не говори «готово».
